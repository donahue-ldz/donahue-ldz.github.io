适配器模式Adapter

By donahue

Aug 16 2014 更新日期:Mar 7 2015

文章目录

大胆的说”适配器的本质就是:转化匹配,尽最大程度去复用已经存在的功能,或者对已经存在的功能进行一定的改装”转换匹配是手段,复用功能才是目的.

首先还是简单的引入一个案例:

`
public class LogModel implements Serializable{

接下来是接口的实现类

此时我们应该咋办呢?
下面是几种可能的解决方案:
1.重新写一个实现类(此案例中只是简单的例子,真实的例子可不是如此简单,加上一些工期限制成本等…)
2.适配器模式适配(适配器的本质就是:转化匹配,复用已经存在的功能,或者对已经存在的功能进行一定的改装)

现在我们就开始讲解适配器模式:
适配器模式的定义:
将一个类的接口转换成客户希望的另外一个接口.适配器模式使得原本由于接口不能兼容而不能工作的那些类可以一起工作.
应用适配器模式解决问题的思路:定义一个类来实现新的(用户新提出定义的)接口,然后在内部实现的时候转调已经存在(大致能满足新接口的功能)的功能,可以再次基础上增加相关的转化实现.要调用原来存在的接口方法方法又有下面两种:
1.对象的组合 :依靠对像的组合来实现的适配器称为对象适配器
2.继承:采取多重继承对一个接口个另外一个接口进行匹配 这种适配器称为 类适配器

其实我们知道新的接口和原来存在的接口之间至少有两个功能是相同的只是方法名不同(当然实际的过程中可能参数,类型,甚至什么都没有关系),仔细思考另外几个方法也能在原来接口的简单包装下能快速实现(添加一定的转化)
先讲解对象的组合方式:简单的代码如下

`
public class Adapter implements Target{
private adaptee adaptee //此时我们将原来的接口称为被适配接口,用adaptee表示,新的接口用Target表示
public Adaptor(Adaptee adaptee){
this.adaptee = adaptee;
}

public list serachLog(String serach){
listlist = adaptee.readLog();
listresult = new list<>();
for(LogModel it:list){
if(it == serach)
result.add(it);
}
return result;
}
public void addLog(listlist){
adaptee.writeLog(list);

}
public void deleteLog(listlist){
listlistTmp = adaptee.readLog();
//

}
public void updateLog(listlist){
listlistTmp = adaptee.readLog();
for(LogModel tmp :listTmp){
for(LogModel it:list){
if(it.getId()==tmp.getId()){
listTmp.remove(tmp);
listTmp.add(it);
}
}
}
}
//…other
}
只是简单的增加一些细节转化,绝大部分是依赖原来的实现这样比新开发的要可靠很稳定,实惠,这样我们就将原来的接口给适配成可以新定义的了 客户端调用代码的化只需要adapter传入相应的就好
public class Client{
Adaptee adaptee = new adapteeImp();
Target target = new Adapter(adaptee);

//do yourFunction by target…
}
`
一般:
Client:调用自己需要的领域接口Target
Target:根据客户端需要建立的领域模型
Adaptee:原来存在的接口,被适配的对象
Adapter:适配器,把Adptee适配成客户端需要的Target

适配器模式的调用图:
Client——-Target(Interface)———Adapter(Impl for Target)
Adapter:适配adaptee,一般使用组合方式(java不支持多继承,但是可以多个implements嘛)

适配器模式的实现:
1.常见实现:适配器类实现Target接口,在实现的时候调用的是adaptee类的现成的方法,所以说适配器类是一个Target类型
2.智能适配器:适配器在完成Target接口给定的功能的时候,不是直接调用adaptee的方法就可以了,自己还加入了很多方法和实现,只是依赖adaptee来做的高层实现
3.适配多个adaptee:实现Target功能的时候调用多个模块多个adaptee对象
4.缺省适配:就是为Target接口提供一个缺省的实现,其他的实现可以extends这个缺省实现,选择重写相关的方法就可以了.

**双向适配
双向适配的含义是将Target适配成Adaptee,把Adaptee适配成Target,前面适配的是后者,但是存在一个潜在的问题就是适配之后就Adaptor就不能再次使用Adaptee接口里面的方法,而只能使用的是Target里面的方法.进行了双向适配之后我们就能在使用Target和Adaptee的场景都能使用Adaptor了呗
实现是比较简单的,只需要在适配器Adaptor里面持有两个对象Adaptee和Target的接口对象就可以了

何时选用适配器模式:**
 
1.想使用一个已经存在的类,但是他的接口不符合你的要求,你可以通过适配来转换
2.想创建一个可以复用的类,这个类可能和一些不兼容的类一起工作,需要什么就适配什么
3.你如果想使用一些已经存在的子类的时候,但是不能对每一个子类去进行”适配”,此时你可以去适配他们的父类,通过父类去调用
