    • 在ubuntu下安装和配置java开发环境，下面对自己的一些实践做了一下总结。
    Sun JDK的安装一般有两种方式：

    ubuntu在其包仓库里都包括有JDK的安装，只有sources.list设置正确（添加正确的源路径），通过apt-get、aptitude、ubuntu软件中心等都能安装，而且相关的配置操作也容易得多。在ubuntu的新发布版中一般都自带了JDK的安装支持，不过版本可能不是最新的。这种安装方法安装的JDK版本不是很灵活，如果你的项目中需要更高版本的JDK或特定版本的JDK，可能就不行了，就要自己重新安装和配置。

    这种安装可自由选择JDK版本，能进行灵活的配置。事实上，JDK的配置也不难。下面就来具体操作吧。
    1. http://www.oracle.com/technetwork/java/javase/downloads/上下载相应版本的JDK环境，这里我使用的是jdk 7u_45，即jdk-7u45-linux-x64.tar.gz2. 我一般把开发软件工具安装到/usr/tools下。
    采用命令行模式将下载的软件包移到你的安装目录下然后解压
    这是我的仅供参考：
    sudo mv jdk-7u45-linux-x64.tar.gz /usr/tools sudo tar -zxvf jdk-7u45-linux-x64.tar.gz

    1. 设置环境变量
      在/etc/profile中添加JDK配置信息：

    2. java 配置

    3. set jdk environment

    4. export JAVA_HOME=/usr/tools/jdk1.7.0_45
    5. export JRE_HOME=/usr/tools/jdk1.7.0_45/jre
    6. export CLASSPATH=.:$CLASSPATH:$JAVA_HOME/lib:$JRE_HOME/lib
    7. export PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin

    8. （让刚刚的配置生效）

    9. source /etc/profile

      注意修改为你的对应的位置和文件名
      配置环境时有几点是要注意的：
      在配置CLASSPATH、PATH时要留心，如果配置的有问题，可能会出现进不了桌面的现象。如，把CLASSPATH路径中的“.:CLASSPATH”写错，写成“:CLASSPATH”或其他就会出问题。出现问题不要紧，按下Ctrl+Alt+F1进入命令行模式，将对应的配置改正确，然后再Ctrl+Alt+F7回到桌面就OK了。
      还有一点就是，配置环境时，环境名称和对应的环境路径之间不要留空格，如JAVA_HOME=/usr/tools/jdk1.7.0_45
      4. 由于ubuntu下自带了gij的JVM实现，所以在终端键入“java -version”时会显示：java version “1.6.0”……等相关信息。这是说系统中已经有了JVM实现，而且gij JVM是默认的JVM实现（被其他的ubuntu工具或软件使用）。这时，就要使用Debian提供的“update-alternatives”工具来完成多版本实现的选择：(要是以上信息未能出现也是正常的)
      sudo update-alternatives —display java
      发现系统只列举了gij JVM；因为jdk7是手动解压安装的。
      sudo update-alternatives —display java
      发现系统只列举了gij JVM；因为jdk7是手动解压安装的。（或者啥都没有列举）继续输入下面的命令：

      sudo update-alternatives --install /usr/bin/java java /usr/tools/jdk1.7.0_45/bin/java 50 sudo update-alternatives --install /usr/bin/javac javac /usr/tools/jdk1.7.0_45/bin/javac 50 sudo update-alternatives --install /usr/bin/javah javah /usr/tools/jdk1.7.0_45/bin/javah 50 sudo update-alternatives --install /usr/bin/javaws javaws /usr/tools/jdk1.7.0_45/bin/javaws 50 sudo update-alternatives --install /usr/bin/javap javap /usr/tools/jdk1.7.0_45/bin/javap 50

    其实只要设置java、javac也是OK的。行尾的50是优先级。
    ls -l /etc/alternatives/java
    发现JVM已经指向了JDK7的解压目录
    sudo update-alternatives --config java 选择刚刚安装的JDK
    java -version　发现已经是…
    其实只要设置java、javac也是OK的。行尾的50是优先级。
    到此，JAVA开发环境平配置完成，可以开始你在ubuntu上的java开发了，GO。
